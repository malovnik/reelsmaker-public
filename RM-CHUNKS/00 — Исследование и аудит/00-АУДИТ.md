# Этап 00: Исследование и аудит проекта

> Статус: ⬜ Не начат
> Родитель: [[PIPELINE-НАВИГАТОР]]
> Проект: **videomaker-рефакторинг**

## Суть этапа

Перед любым рефакторингом — картография. Мы должны точно знать: какие backend-сервисы живые, какие мёртвые, где прячется PRO-профиль, что в settings-страницах пересекается, как устроены текущие настройки проектов, какие pipeline-stages вызываются и в каком порядке, и где ключевые UX-боли, о которых говорил владелец.

Этап чисто **исследовательский**: ничего не удаляем, не меняем. Только документы в `docs/audit/` + memory Serena.

**Режим работы:** Sequential. Каждый чанк АУД-XX = отдельный документ по одной теме аудита.

## Подэтапы

Этап состоит из 7 подэтапов (REFACTR-00..REFACTR-06):

- **REFACTR-00** — Карта backend-сервисов (что живое/мёртвое среди 80+ модулей в `services/`) ⬜
- **REFACTR-01** — Карта frontend-страниц и компонентов (все `app/*` и `components/*`) ⬜
- **REFACTR-02** — Инвентаризация настроек (все разделы `/settings/*`) ⬜
- **REFACTR-03** — Профили и PRO-код (точки удаления) ⬜
- **REFACTR-04** — Схема данных (SQLite + файловое хранилище) ⬜
- **REFACTR-05** — Pipeline stages и зависимости ⬜
- **REFACTR-06** — UX-боли: OOM, поиск Cmd+K, горизонт. прокрутка ⬜

## Вход

```markdown

<таблица: имя модуля — назначение — статус (живой/мёртвый/дубликат) — ссылки>

## Ключевые находки
<5-10 пунктов, каждый с цитатой/ссылкой на конкретный файл:строка>

## Риски при рефакторинге
<что нельзя сломать>

## Рекомендации для следующих этапов

```
