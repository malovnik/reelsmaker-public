## Роли

### R-AUDITOR — Аудитор проекта
**Профессия:** Senior-разработчик-картограф.
**Soul:** Видит наслоения «накидали-накидали» в компонентах. Отличает живое от мёртвого, копипаст от переиспользования.

### R-FRONTEND-ARCHITECT — Фронт-архитектор (консультативно)
**Профессия:** Senior React.
**Soul:** Думает о миграции уже на этапе аудита — чтобы не потерять ни одной страницы при переходе Next.js → Vite.

---

## ТРИЗ-принцип

*Принцип динамичности.* Аудит должен быть ориентирован на миграцию: каждая страница получает метку «переносится как есть» / «переделывается на новом дизайне» / «удаляется». Если аудит этого не даёт — этап 04 не сможет стартовать.

### 01.4 Выявить проблемные места

Владелец назвал конкретные боли — каждую подтвердить цитатой из кода:

- [x] `NODE_OPTIONS=--max-old-space-size=12288` → `package.json:6`.
- [x] Горизонтальная прокрутка в Subtitles → найти CSS/JSX.
- [x] Пост-продакшн без структуры → какие компоненты, какие секции.
- [x] Смешанные тёмные/светлые шрифты → поиск хардкод-цветов (`text-white`, `text-black`, `#fff` в inline).
- [x] Cmd+K не работает → проверить наличие хэндлера в `TopBar.tsx` или глобальном listener.
