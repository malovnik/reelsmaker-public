# Этап 02: Бэкенд — чистка и проекты

> Статус: ⬜ Не начат
> Родитель: [[PIPELINE-НАВИГАТОР]]
> Проект: **videomaker-рефакторинг**

## Суть этапа

Хирургия бэкенда. Четыре направления:

1. **Удалить PRO-профиль** полностью (код, storage, UI-хуки, миграция данных пользователей).
2. **Переделать модель проекта**: snapshot настроек внутри проекта, маркеры стадий pipeline, поддержка «перезапустить с шага X».
3. **Добавить автосохранение + копирование настроек** из предыдущих проектов.
4. **Ввести сущность «идея рилса»**: описание, хук, текст, статус (pending/approved/rejected/regenerating) + API approve/reject/regenerate.

Всё делается без остановки работы существующих функций. Миграции Alembic, обратная совместимость, никаких mocks.

**Режим работы:** Sequential. Каждый чанк — одна хирургическая операция.
