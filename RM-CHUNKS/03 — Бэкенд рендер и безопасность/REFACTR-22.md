## Микрозадачи

### 22.1 Теория VideoToolbox bitrate control

- [ ] Context7 / ffmpeg docs:
  - `hevc_videotoolbox` поддерживает `-b:v` (bitrate) + `-maxrate` + `-bufsize`.
  - CRF-подобное через `-q:v` (0-100, где выше = качественнее). На VideoToolbox интерпретация другая.
- [ ] Зафиксировать рекомендуемые параметры для 9:16 короткого рилса (30 сек — 2 мин).

### 22.2 Экспериментальный прогон

- [ ] Взять тестовый 30-сек ролик 9:16.
- [ ] Рендерить с 3 конфигурациями:
  - A: `-b:v 15M -maxrate 18M -bufsize 30M`.
  - B: `-b:v 15M -maxrate 20M -bufsize 30M -q:v 55`.
  - C: `-b:v 12M -maxrate 20M -bufsize 30M -q:v 60` (попробовать меньше avg, больше peak).
- [ ] Сравнить: размер файла, визуальное качество (глаз), SSIM (если есть возможность).

### 22.3 Выбрать профиль

Профиль по умолчанию для Viral 2026 — тот, что даёт ≥15 Mbps target при минимальном размере с визуально приемлемым качеством.

Записать в `services/performance_settings_store.py` как `render_profile = "viral_2026_hevc_15"`.

### 22.4 Настройки override

Пользователь может override через `/settings/performance`:
- bitrate_mbps (default 15)
- max_bitrate_mbps (default 18)
- quality_q (default 55)
- preset (default medium, hardware игнорирует, но у software учитывается)

### 22.5 Документ

`docs/performance/render-profiles.md`:
- Таблица профилей (name, params, typical bitrate, typical size, use-case).
- Результаты эксперимента.
