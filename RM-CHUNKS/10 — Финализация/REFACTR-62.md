## Микрозадачи

### 62.1 Сценарий

1. Запустить `./run.sh`.
2. Открыть `http://localhost:3000/` в браузере.
3. Студия пуста (чистая БД) → Empty state.
4. Клик «Новый проект» → модалка.
5. Dropzone тестового видео (60 секунд, 9:16) → upload с прогрессом.
6. Название автозаполнилось.
7. Выбор «С нуля» (Viral 2026).
8. Клик «Создать» → навигация в Workbench.
9. Workbench: video player показывает загруженное видео.
10. Pipeline timeline: transcription стартует автоматически.
11. Статусы обновляются через SSE.
12. Транскрипция завершается → silence cut → multi-pass LLM → идеи генерируются.
13. Idea grid в main area заполняется (N идей).
14. Approve 3 идеи, reject 1, regenerate 1 с custom prompt.
15. Клик «Рендерить одобренные» (3).
16. Clips tab → прогресс каждого клипа.
17. Готовые клипы → preview в lightbox → download одного.
18. Возврат в Студию → видим проект в grid.
19. Контекстное меню → «Открыть в Finder» → папка открывается.
20. Переключение на Light тему → работает.

### 62.2 Реализация (опции)

**Опция A: Playwright E2E script**
- Если есть ресурс — `apps/e2e/` с Playwright-тестом.
- Run: `pnpm e2e`.

**Опция B: Ручной чек-лист**
- `docs/e2e/SMOKE-1-NEW-PROJECT.md` — markdown-чеклист с 20 пунктами.
- Галочки заполняются при ручном прогоне.

Рекомендация: **A если время позволяет, B как минимум**.

### 62.3 Thresholds

- Total time от upload до готовых 3 клипов (60-сек видео): **≤3 минут**.
- Никаких console.error.
- Никаких 500-ответов backend.
- RAM peak dev-сервера: <1 ГБ.
