# FA-3 — Аудит дружелюбности интерфейса

Роль: Conversation/UX-Writing & Onboarding Specialist.
Область: `/apps/frontend/src` (Vite + React Router, русскоязычный UI).

---

## Вердикт по онбордингу

**Явного онбординга нет.** Нет first-run flow, нет приветственного тура, нет чеклиста «с чего начать», нет пустого-состояния-как-онбординга на уровне всего приложения. Новый пользователь попадает сразу на дашборд (`HomeClient`) с формой загрузки.

**Что спасает ситуацию (мягкий, «встроенный» онбординг):**
- `DashboardHero.buildGreeting()` — живое приветствие по времени суток («Утро. Что нарезаем?»), осознанно уходит от генерик-фраз банкомата. Тон отличный.
- Wizard сам по себе работает как пошаговый онбординг: 6 пронумерованных шагов (`Step index=1..6`) с заголовками-вопросами («Под какой тип видео нарезаем», «Видео», «Формат и количество»).
- Шаг 1 имеет дружелюбный hint: «Если не уверен — выбери любой. После загрузки мы подскажем точный профиль.» — точно в духе «ведём за руку».
- Empty state списка нарезок (`JobList`, 0 джобов) ведёт к действию и снимает тревогу: «Загрузи первое видео в форме выше — первая нарезка займёт около двух минут».
- При отсутствии LLM-ключа `UploadWizard` показывает понятную инструкцию что сделать (`GEMINI_API_KEY` в `.env`, `./run.sh`).

**Главный пробел для будущих режимов:** в коде нет инфраструктуры под заявленные «Эксперт-режим» и «Пошаговый режим». Нет переключателя режима, нет state-флага `expertMode`/`firstRun`/`hasSeen`, нет компонента Tooltip (см. ниже). Эти режимы придётся строить с нуля.

Оценка онбординга: **6/10** — неплохой имплицитный онбординг через wizard и тексты, но нет ни одного явного механизма для новичка и нулевая инфраструктура под планируемые режимы.

## Покрытие контролов подсказками (%)

Разделяю по зонам, т.к. покрытие очень неравномерное.

| Зона | Контролов (прибл.) | С подсказкой | Покрытие |
|------|------|------|------|
| Settings / Performance groups (`performance-groups/*`) | ~80 | ~80 | ~100% |
| Settings / Post-production (`post-production/*`) | ~25 | ~25 | ~100% |
| Upload Wizard (основные шаги) | ~18 | ~16 | ~90% |
| Job / Reel / Tinder экраны | ~20 | ~6 (только icon `title=`) | ~30% |
| Scheduler / Accounts / Projects | ~30 | ~8 | ~25% |
| Навигация (NavRail / TopBar) | ~10 | 0 | 0% |

**Взвешенная оценка покрытия: ~70%.** Тянет вверх образцовый settings-слой.

Сильная сторона: shared-примитивы `SwitchRow`, `NumberRow`, `SliderRow`, `SelectRow` делают `hint: string` **обязательным пропом** (не optional) — значит каждый контрол в настройках физически не может появиться без подсказки. Это лучшая практика и готовый фундамент для Эксперт-режима. Тексты подсказок содержательные, объясняют не «что», а «зачем и какой эффект» (см. `JLCutGroup`: «L-cut: аудио текущей сцены ещё звучит поверх видео следующей… делает рилс менее „резаным"»).

Слабая сторона: вне настроек подсказки держатся только на нативном HTML `title=` на icon-кнопках (touch-устройства их не видят), а навигация и большинство кнопок действий (экспорт, тиндер, публикация) не подсказаны вовсе.

**Критично не хватает подсказок:**
- Навигация `NavRail`/`TopBar` — пункты меню без пояснений (что за «Тиндер», «Шедулер»).
- Кнопки действий в Job/Reel экранах (`ExportDialog`, `TinderClient`, `ManualPublishButton`) — только icon-`title`.
- В wizard у `<select>` «Проект (папка)», «LLM-провайдер», «Распознавание речи» — провайдеры показаны сырыми машинными именами (`gemini`, `zhipu`, имена транскрайберов как есть), без человеческой расшифровки.

## Топ проблем дружелюбности

### FA3-01 — Нет инфраструктуры под Эксперт/Пошаговый режим — HIGH
Где: всё приложение. Нет компонента Tooltip, нет state-флага режима, нет персистенции «новичок/эксперт».
Описание: заявленная цель (подсказка напротив 100% контролов + пошаговый режим) сейчас недостижима — нет переиспользуемого Tooltip-компонента (есть только один кастомный `ProfileTooltip` и нативные `title=`), нет глобального контекста режима.
Рекомендация: создать `<Tooltip>`/`<InfoHint>` компонент (hover + focus + tap для мобильных), вынести `hint` из settings-примитивов в универсальный паттерн, добавить `UiModeContext` (`stepByStep | expert`) с персистенцией в localStorage. Settings-слой уже даёт готовую модель: `hint` как обязательный проп.

### FA3-02 — Сырые технические ошибки утекают к пользователю — HIGH
Где: `upload/useWizardState.ts:355` (`Ошибка ${err.status}: ${JSON.stringify(err.detail)}`), `PromptsEditorClient.tsx:64`, `SubtitleSettingsClient.tsx:454`, `VisionProfilesSettingsClient.tsx:171,192`, `post-production/shared.tsx:125`, и ~цепочка `setError(String(err))` в ~8 местах. Также `JobHero.tsx:116` и `UploadWizard.tsx:889` выводят `job.error`/`sse.lastEvent.error` напрямую из бэкенда.
Описание: пользователь видит `Ошибка 500: {"detail":...}`, JSON-дампы и потенциально английский текст из бэкенд-исключений. Это пугает новичка и нарушает требование «только русский, подсказывай решение».
Рекомендация: единый хелпер `humanizeError(err)` — маппинг по `err.status` (400→«Проверь введённые данные», 413→«Файл слишком большой», 5xx→«Что-то сломалось на нашей стороне, попробуй ещё раз») с fallback на `extractDetail`, без `JSON.stringify`. Технический detail — только в `console`/раскрываемом блоке, не в основном тексте.

### FA3-03 — Нет тост-системы; обратная связь несистемна — MEDIUM
Где: всё приложение (grep: 0 совпадений `toast`/`sonner`/`Toaster`).
Описание: успех/ошибка показываются либо инлайн-div'ами, либо вообще без подтверждения. Успешное переименование, сохранение настроек, удаление — без видимого «Готово». Деструктивные действия используют нативный `window.confirm` (13 мест) — он не стилизован, выпадает из дизайна, и текст в нём (хоть и русский) выглядит как системный алерт.
Рекомендация: ввести лёгкую тост-систему для успехов/ошибок действий; заменить `window.confirm` на стилизованный диалог подтверждения (особенно для «Удалить полностью — необратимо»).

### FA3-04 — Машинные имена в селекторах wizard — MEDIUM
Где: `UploadWizard.tsx:619-639` (Распознавание речи, LLM-провайдер).
Описание: опции рендерятся сырыми value (`gemini`, `zhipu`, технические имена транскрайберов) без человеческих лейблов — в отличие от соседних селектов (язык, кадрирование, профиль), где маппинг лейблов сделан образцово.
Рекомендация: добавить `LABELS` мапы как у `SOURCE_LANG_LABELS`/`FIT_MODE_LABEL` («Gemini», «Zhipu GLM», «Whisper (локально)» и т.д.).

### FA3-05 — Навигация без подсказок — MEDIUM (важно для Эксперт-режима)
Где: `components/shell/NavRail.tsx`, `TopBar.tsx`.
Описание: пункты навигации не имеют пояснений. Новичок не понимает разницу «Тиндер» vs «Проекты» vs «Шедулер» до клика.
Рекомендация: tooltip/подзаголовок на каждый пункт; при collapsed-rail — обязательный tooltip с названием.

### FA3-06 — Полу-английские/жаргонные тексты в части блоков — LOW
Где: `WizardSteps.tsx` — «Стиль монтажа (composer strategy)», «Plotline-стиль», «cross-context рисков», «ranked-сегментов», «advisor'а»; `UploadWizard.tsx:476` «companion», «split-screen»; `UploadWizard.tsx:560` «system-prompt всех LLM-вызовов (12 агентов)».
Описание: для эксперта ок, для новичка — шум из англицизмов и внутренней терминологии. Нарушает «без английского».
Рекомендация: в пошаговом режиме прятать/упрощать; термины давать через tooltip, а не в основном тексте. «companion» → «второе видео для сплит-скрина».

### FA3-07 — `title=` как единственная подсказка не работает на тач — LOW
Где: все icon-кнопки с `title=` (JobCard, ReelCard, ReelGrid, BulkActions в `JobList`, HeatmapBar и т.д.).
Описание: на мобильных `title` не показывается — мобильный пользователь не получит подсказку вовсе. CLAUDE-требование «сразу прорабатываем мобайл».
Рекомендация: заменить на единый `<Tooltip>` (см. FA3-01) с tap-поведением.

## Что сделано хорошо (для контекста, не править)

- **Empty states — образцовые и единообразные.** Везде «нет данных + что сделать»: `JobList` («Загрузи первое видео…»), `ProjectsList:33` («Создай первый проект — он станет папкой…»), `PresetListColumn:32` («Пресетов пока нет — жми „+ новый", и через минуту будет первый»), `ReelPicker:155-159`, `ProjectFolderPage:86-87`, `CaptionsEditor:71`. Тон тёплый, ведёт к действию.
- **Прогресс/лоадеры — сильно.** `PipelineTimeline` — пошаговый таймлайн с русскими лейблами стадий («Поиск лучших моментов», «Сборка рилсов»), живым пульсом активной стадии, ✓ и длительностью завершённых, отдельным error-маркером. `JobHero` — цветной прогресс-бар (danger/success/accent), `aria-live` на счётчике символов в wizard. SSE прогресс с человеческим `message`.
- **Тон приветствий и микрокопирайта** — живой, без клише, в характере продукта (DashboardHero, «Робот-монтажёр проанализирует дорожку и сам примет решения»).
- **Settings hint-тексты** — лучшая часть продукта: объясняют эффект решения, дают рекомендуемое значение, диапазон. Готовая база для Эксперт-режима.
- **Большинство ошибок валидации форм — на русском и подсказывают решение** («Введи название — без него пресет не сохранится», «Лого больше 1.5 МБ — сожми перед загрузкой»).
