# D2 — Режим «Пошаговый» (Stepwise / Guided Flow)

> Спека режима для новичка. Линейная цепочка «один экран = один шаг» от крупной кнопки «Создать проект» до готовых рилсов.
> Роль автора: Senior Product Designer (Onboarding & Guided Flows). Принцип: веди за руку, один выбор за раз, антиперегруз.
> Это **спецификация**, не код. Визуальный язык — брендбук Никиты Малова: латунь на чёрном (`#C9A84C` на `#0A0A0A`), прямые углы (`border-radius: 0`), шрифты Noto Serif JP (заголовки) / Manrope (текст) / JetBrains Mono (мета, теги), пиксельный тег Press Start 2P.

---

## 0. Философия режима

Эксперт-режим (`/settings/performance` + длинный скролл-визард) показывает 20+ групп параметров сразу — это панель управления для того, кто знает систему. Пошаговый режим решает обратную задачу: **новичок не должен видеть больше одного решения за раз**.

Три правила, которым подчинён каждый экран:

1. **Один экран — один логический выбор.** Не «настрой 6 вещей», а «выбери одно, жми Далее».
2. **Дефолт уже выбран и подсвечен.** Любой шаг можно пройти кнопкой «Далее», ничего не трогая. Это и есть Auto-режим (см. §4).
3. **Из любого шага видно, где ты и сколько осталось.** Прогресс-полоса сверху, навигация назад/далее снизу.

Цепочка покрывает живое ядро бэкенда (BACKEND-MAP §«Живое ядро»): ingest → narrative → ffmpeg-рендер → SSE → разметка. Фиктивные и dormant-возможности (chaptered, tier «pro» как реальная модель, vision по умолчанию) в пошаговом режиме **скрыты** — новичку их не показываем (см. §5 «Что намеренно спрятано»).

## 1. Карта шагов (overview)

```
[ СТАРТ ]  Создать проект                    ← крупная кнопка, primary золотая обводка
    │
    ▼
S1  Проект            создать / выбрать папку
S2  Видео             загрузка файла (drag & drop)
S3  Вид рилсов        narrative_mode человеческими именами
S4  Субтитры          стиль + вид (пресет ASS)
S5  Обработка         что включить (тумблеры, всё OFF по умолчанию)
S6  Модели            локально vs облако (STT + LLM-провайдер + tier)
S7  Запуск            сводка → большая кнопка «Запустить»
S8  Прогресс          live SSE, стадия + %, кнопка «Отменить»
S9  Результаты        галерея рилсов (ReelGrid + Heatmap)
S10 Разметка          выбор лучших (Tinder / like-dislike)
S11 Экспорт / публикация
```

Шаги S1–S6 — настройка (можно листать назад/вперёд свободно). S7 — точка невозврата (после запуска параметры заморожены). S8–S11 — результат.

**Сквозной прогресс-индикатор** (один компонент на всех экранах настройки):

```
НАСТРОЙКА                                          ШАГ 3 / 6
██████████░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░
 ▲                ▲
 пройдено      текущий (золотой)
```

- Полоса: фон `#1A1A1A`, заполнение `#C9A84C`, высота 4px, прямые углы.
- Подпись слева: JetBrains Mono 11px uppercase `#8A8278`. Счётчик справа: `#C9A84C`.
- Под полосой — кликабельные метки пройденных шагов (можно вернуться тапом).

## 4. Как реализован Auto-режим для новичка

Auto-режим — не отдельный экран, а **сквозное свойство всей цепочки**: «просто сделай за меня». Реализуется тремя слоями:

1. **Дефолт на каждом шаге уже выбран и подсвечен.** Новичок может пройти S1→S7, нажимая только «Далее», ни к чему не прикасаясь. Получит: авто-имя проекта → режим «Режиссёрский» → пресет «Акцент» → обработка по умолчанию (только громкость) → локальный STT + Gemini + Эконом. Это «дефолтный прогон» бэкенда (BACKEND-MAP §«Default profile»).

2. **Скрытая авто-конфигурация на запуске.** На S7 кнопка «ЗАПУСТИТЬ» вызывает `POST /jobs/{id}/auto-analyze` и **сразу применяет** результат (`PATCH /auto-config` с 15 параметрами pacing/zoom/coherence) — новичку этот экран approve **не показывается**. Бэкенд сам подбирает темп/зум/порог когерентности под конкретное видео. (В Эксперт-режиме тот же auto-analyze показывает `AutoConfigSummary` card с возможностью править — здесь он молчаливый.)

3. **Жёсткое сокрытие сложности.** Всё, что может перегрузить (15 параметров auto-config, face-tracking, vision, multi-arc, split-screen, J/L-cuts, выбор composer-стратегии, chaptered-режим, ручной approve), в пошаговом режиме **не существует на экране**. Единственная дверь к ним — ссылка «продвинутые настройки →». Новичок физически не может сломать прогон лишним тумблером.

**Формула Auto-режима:** разумные дефолты (видны) + авто-конфиг под видео (скрыт) + отсечение сложности (спрятано за одной ссылкой). Результат — путь «загрузил → нажал Далее 5 раз → Запустить → получил рилсы» без единого осознанного технического решения.

## 5. Переключение в Эксперт-режим + что намеренно спрятано

**Точки входа в Эксперт-режим** (ссылка «продвинутые настройки →», `#C9A84C`, моноширинная):
- На СТАРТ-экране (под кнопкой «Создать проект») — для тех, кто уже знает систему.
- На S5 («Тонкая ручная настройка →») — переход в `/settings/performance` и `/settings/post-production`.
- На S6 — переход в `/settings/models`.

Переключение **сохраняет уже введённые данные** (проект, файл, выборы) и открывает длинный скролл-визард / панель `/settings/*` с тем же контекстом. Обратно — кнопка «Вернуться к пошаговому» в шапке Эксперт-режима.

**Намеренно спрятано от новичка** (доступно только в Эксперте):

| Спрятано | Причина |
|---|---|
| `chaptered`-режим | author-marked broken, выбор ломает прогон |
| Ручной approve auto-config (15 параметров) | перегруз; в Auto применяется молча |
| Face-tracking, vision, cover-selector | dormant / OFF по умолчанию, жгут CPU |
| Multi-arc, split-screen, J/L-cuts, breath, composer strategy | продвинутый DSP, не для первого прогона |
| Реальная семантика tier «Pro» | фикция (всё Flash-Lite) — на S6 честно предупреждаем, но не даём ложного выбора |
| Legacy `/schedule` + `ScheduleButton` | дублирует Publer, путает (обрыв #5) — ведём только через `CampaignWizard` |
