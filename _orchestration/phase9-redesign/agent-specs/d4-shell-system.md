## 0. Токены и инварианты (вход от D-токенов, не дублировать)

Оболочка использует существующие OKLCH-токены. Маппинг брендбук → текущие переменные:

| Брендбук | Роль | Текущая переменная |
|----------|------|--------------------|
| Kuro `#0A0A0A` | фон страницы | `--ink` |
| Sumi `#1A1A1A` | карточки, рейл, второй слой | `--ink-2` |
| Hai `#2A2A2A` | бордеры, разделители | `--line`, `--line-soft` |
| Kinzoku `#C9A84C` | главный акцент (лого, CTA, active) | `--gold` |
| Dō `#B87333` | вторичный акцент (мета-теги, иконки) | `--copper` (добавить, если нет) |
| Shiroi `#F0E6D2` | основной текст | `--paper` |
| Kasumi `#8A8278` | мета-текст, неактив | `--mute`, `--mute-2` |
| Chi `#8B2500` | деструктив-акцент (точечно) | `--danger` (добавить) |

**Шрифты (уже self-hosted):** display = Geist/Noto Serif JP роль (`--font-display`), body = Inter (`--font-sans`), service caps = JetBrains Mono (`--font-mono`). Лого в шапке — display-serif золотом, навигация и коды разделов — mono uppercase, как в брендбуке.

**Правила, унаследованные от брендбука:**
- `border-radius: 0` или минимальный (4px) на структурных элементах — прямые углы.
- Акцент `--gold` ≤ 20-30% площади. Никаких градиентов из акцентов.
- `--danger` (Chi) — точечно: одна линия / иконка / бордер модалки, не заливка.
- Запрещены: параллакс, бесконечные rotation, bounce/elastic, анимации > 1s.

## 1. ГЛОБАЛЬНАЯ НАВИГАЦИЯ — единая структура

### 1.1 Проблема (U-02)

Сейчас два конфликтующих источника навигации:
- `NavRail` (9 пунктов) выносит 6 из 8 settings-разделов как top-level вперемешку со Студией/Проектами/Шедулером. `brand` и `maintenance` в рейл не попадают — **недостижимы из рейла**.
- `SettingsSubNav` (8 пунктов) — полный список настроек, живёт внутри `SettingsLayout`.

Один и тот же раздел («Модели», «Субтитры»…) появляется в двух местах с разными `code` и подсветкой → пользователь не понимает, где он.

### 1.2 Решение: двухуровневая модель «3 рабочие зоны + Настройки»

Навигация рейла = **только верхнеуровневые рабочие зоны продукта**. Настройки — один пункт, раскрывающийся в свою суб-навигацию внутри `SettingsLayout`. `SettingsSubNav` остаётся **единственным** источником разделов настроек.

**Структура рейла (NavRail) — 4 пункта:**

| code | label | href | matchPrefix | зона |
|------|-------|------|-------------|------|
| `STU` | Студия | `/` | (точное `/`) | создать нарезку, последние работы |
| `LIB` | Проекты | `/projects` | `/projects`, `/jobs` | библиотека работ, папки, клипы |
| `PLN` | Планировщик | `/scheduler` | `/scheduler` | автопостинг, аккаунты, кампании |
| `CFG` | Настройки | `/settings/profiles` | `/settings` | все 8 разделов настроек |

Пункт «Настройки» в рейле активен для всего `/settings/*`. Внутри `/settings/*` рендерится `SettingsSubNav` со всеми 8 разделами (включая ранее недостижимые `brand`, `maintenance`) — это и закрывает U-02: настройки больше не дублируются в рейле, у них одна точка входа.

**Единый словарь разделов** (`lib/nav/routes.ts`) — один источник правды для рейла, суб-нава, хлебных крошек и онбординга. Устраняет рассинхрон `code`/`label` между NavRail и SettingsSubNav.

```
├─ LIB  Проекты           /projects
│        └─ папка/клипы    /projects/:id/folder · /jobs/:id · /jobs/:id/reels/:reelId · /jobs/:id/tinder
├─ PLN  Планировщик        /scheduler
│        └─ аккаунты/кампании /scheduler/accounts · /scheduler/new · /scheduler/campaigns/:id · /scheduler/presets
└─ CFG  Настройки          /settings  (→ SettingsSubNav)
         ├─ PRF Профили нарезки      /settings/profiles
         ├─ MDL Модели               /settings/models
         ├─ PFM Производительность   /settings/performance
         ├─ CAP Субтитры             /settings/subtitles
         ├─ POP Пост-продакшн        /settings/post-production
```

1. **Бургер** — только `<lg` (открывает drawer; паттерн уже работает, сохранить).
2. **Лого** (на desktop, где рейл уже виден лого можно не дублировать — лого живёт в шапке рейла; на mobile, где рейл скрыт, лого показывается в шапке). Лого = display-serif «Reelibra» цвет `--gold` + mono-бейдж `β`.

| Breakpoint | Рейл | Шапка |

## 2. ПЕРЕКЛЮЧАТЕЛЬ РЕЖИМОВ — Пошаговый / Эксперт

### 2.1 Где живёт

В **шапке справа**, перед индикатором статуса. Глобальный, виден на всех экранах (кроме онбординга) → пользователь всегда знает и может сменить режим. На mobile сжимается до иконки-сегмента с короткими подписями «Шаг / Эксп».

### 2.2 Как выглядит

Сегмент-контрол (две кнопки в общей mono-рамке `border --line`, `border-radius 4px`), не toggle-галочка — оба варианта всегда подписаны, активный залит `--gold` с текстом `--ink`, неактивный — `--mute-2` на прозрачном.

```
┌─────────────────────────────┐
│  ⊞ Пошаговый │ ⌘ Эксперт    │   active = Пошаговый (залит --gold, текст --ink)
└─────────────────────────────┘   inactive = Эксперт (--mute-2, прозрачный, hover → --paper)

mobile:  [ Шаг │ Эксп ]   (та же логика, короткие подписи, иконки-only при <380px)
```

`focus-visible:outline-2 outline-offset-2 --gold`. `aria-pressed` на активной. Смена режима → короткий тост-подтверждение (см. §4.3): «Режим: Эксперт. Показываю все опции сразу.»

### 2.3 Как сохраняется — `UiModeContext` (закрывает FA3-01)

```
UiModeProvider (оборачивает RootLayout, выше Outlet)
  state: mode: "guided" | "expert"
  default: "guided"  (новичок-first; см. аудит — первый экран сейчас на эксперта, это и чиним)
  persist: localStorage["reelibra.uiMode"]
  api: { mode, setMode(m), isGuided, isExpert }
  hook: useUiMode()
```

- При первом запуске (нет ключа в localStorage) → `guided`. Онбординг (§3) предлагает выбрать, запись только после явного выбора.
- Чтение синхронно при инициализации (no-flash): прочитать localStorage до первого рендера контента, чтобы не мигало переключением раскладки.
- `<Tooltip>` (тоже отсутствует — FA3-01) вводится этим же слоем примитивов; подсказки показываются всегда, но в Пошаговом — развёрнутые инлайн-хинты, в Эксперте — только по hover/focus иконки «?».

### 2.4 Что меняется при переключении

Режим — это **прогрессивное раскрытие сложности**, не разные функции. Бизнес-логика и роуты идентичны.

| Аспект | Пошаговый (guided) | Эксперт (expert) |
|--------|--------------------|--------------------|
| Студия / создание нарезки | мастер по шагам: 1 источник → 2 профиль → 3 запуск; продвинутые опции скрыты за «Настроить вручную» | одна форма со всеми опциями сразу (текущий экран) |
| Хинты | развёрнутые инлайн-подсказки под каждым полем (что это даёт + сколько займёт) | компактно; подсказка по «?» на hover/focus |
| Машинные имена моделей (FA3-04) | человеческие ярлыки («Быстрая / Качественная»), тех-имя в подсказке | тех-имена видимы (`gemini-2.5-flash`), это норма для эксперта |
| Опции пост-продакшна, промпты, производительность | скрыты по умолчанию, доступны через «Расширенные настройки» | развёрнуты |
| Дефолты | безопасные пресеты применяются молча | всё на виду, ничего не предзадано жёстко |

Режим **не** меняет: набор роутов, данные, доступность разделов настроек (просто степень их раскрытия). Эксперт-режим уже частично существует (упомянут в Обслуживании) — формализуется через контекст.

## 3. ОНБОРДИНГ / FIRST-RUN (закрывает U-10/FA3)

### 3.1 Триггер и принцип

Не навязчивый тур по стрелочкам. Один welcome-экран, показывается когда:
1. `localStorage["reelibra.onboarded"]` отсутствует, **И/ИЛИ**
2. `/health` сообщает о неготовности (`llm_providers` пуст = нет GEMINI_API_KEY; `ffmpeg.available === false`).

Если ключ есть и онбординг пройден — экран не показывается никогда. Закрыть можно всегда («Пропустить» → пишем `onboarded`).

### 3.2 Структура welcome-экрана

Полноэкранный оверлей на `--ink`, по центру карточка `--ink-2` border `--line`, фоновый энсо 8% (как в брендбуке hero). Три смысловых блока, вертикально:

**Блок A — Готовность системы (health-gate).** Чек-лист из `/health`:
- `Ключ Gemini` → ✓ (gold) если `llm_providers` непустой, иначе ✗ (`--danger`) + инлайн-инструкция «Добавьте `GEMINI_API_KEY` в переменные окружения и перезапустите — без него нарезка не запустится».
- `FFmpeg` → ✓/✗ из `ffmpeg.available`.
- `Распознавание речи` → ✓ если `transcribers` непустой.

Кнопка «Проверить снова» перезапрашивает `/health` (без перезагрузки страницы). Пока есть красный пункт — кнопка «Начать» неактивна (с подсказкой почему).

**Блок B — Знакомство с двумя режимами.** Две карточки рядом (на mobile — стопкой), выбор пишет `UiModeContext`:
- «Пошаговый» — «Проведу за руку: источник → стиль → запуск. Опытные детали спрятаны.»
- «Эксперт» — «Все параметры на одном экране. Для тех, кто уже резал.»

**Блок C — Первое действие.** Одна gold-кнопка «Создать первую нарезку» → ведёт в Студию в выбранном режиме. Вторичная ссылка «Осмотреться сам» → закрывает оверлей на Студии.

### 3.3 Не навязчиво

- Никаких пошаговых coachmarks поверх UI.
- Не блокирует, если система готова и онбординг пройден.
- Health-gate показывается **и вне онбординга** — если ключ пропал в рантайне, любой запуск нарезки покажет дружелюбную ошибку через `humanizeError()` (§4.4), а не сырой 500.

## 4. СИСТЕМНЫЕ ПАТТЕРНЫ

Все — новый слой `components/ui/` (примитивы, отсутствуют сейчас — code-health P1). Стилизованы по брендбуку.

### 4.1 Error Boundary (закрывает FA4-P0-01/02)

Сейчас ноль React Error Boundary → любой `throw` = белый экран. Lazy `ProjectFolderPage` в `Suspense` без error-обработки → падение чанка = белый экран.

**Два уровня:**
1. **Root-level** `<AppErrorBoundary>` оборачивает `<Outlet>` в `RootLayout` (внутри shell, чтобы рейл/шапка остались). Ловит рантайм-throw любого экрана.
2. **Route-level** для lazy-чанков: `Suspense fallback` = скелетон (§5), а `errorElement` на lazy-роутах (react-router `errorElement`) ловит rejection чанка с кнопкой «Перезагрузить».

**Экран «что-то сломалось» в самурайском стиле:**
- Фон `--ink`, по центру: пиксельный знак-мон приглушённый, иероглиф 侍 как декор `--copper`.
- Заголовок display-serif `--paper`: «Клинок затупился». Подзаголовок `--mute`: человеческое объяснение без стектрейса.
- В dev-режиме сворачиваемый `<details>` с тех-деталями; в prod скрыт.
- Кнопки: gold «Попробовать снова» (reset boundary) + вторичная «На главную».
- `--danger` точечно — одна тонкая линия под заголовком, не заливка.

```
┌──────────────────────────────────────────┐
│                  侍   (copper, декор)      │
│                                          │
│            Клинок затупился               │  ← display-serif --paper
│   ────────────────────────                │  ← --danger линия, тонкая
│   Экран не смог отрисоваться. Это не вы — │  ← --mute
│   что-то в приложении. Данные целы.       │
│                                          │
│   [ Попробовать снова ]   На главную      │
│                                          │
│   ▸ Технические детали (только dev)       │
└──────────────────────────────────────────┘
```

### 4.2 Пустые состояния — ведут к действию

Принцип (anti-slop): «здесь пока ничего нет» запрещено. Формула = что это + первое действие + оценка времени.

| Экран | Пустое состояние |
|-------|------------------|
| Студия (нет работ) | «Ещё ни одной нарезки. Загрузите видео — первый клип будет готов за пару минут.» + gold-кнопка «Создать нарезку» |
| Проекты (пусто) | «Здесь копятся ваши готовые клипы. Запустите первую нарезку из Студии.» + ссылка в Студию |
| Планировщик (нет кампаний) | «Автопостинг ещё не настроен. Подключите аккаунт — и клипы будут публиковаться сами.» + «Подключить аккаунт» |
| Результат (0 клипов после прогона) | «Из этого видео не вышло клипов. Обычно дело в коротком/тихом исходнике — попробуйте видео подлиннее.» + «Запустить заново» |

Визуально: приглушённый пиксельный знак/иконка `--mute`, заголовок `--paper`, текст `--mute`, gold-CTA. Прямые углы, тонкий border `--line`.

### 4.3 Тост-система (закрывает FA3-03)

`ToastProvider` + `useToast()` в слое примитивов. Контейнер — `fixed` низ-право (desktop) / низ по центру (mobile), `z` выше шапки.

| Тип | Граница/иконка | Поведение |
|-----|----------------|-----------|
| success | `--gold` иконка ✓ | автоскрытие 4s |
| error | `--danger` иконка, `--danger` левый бордер 2px | автоскрытие 8s + кнопка «Подробнее» (раскрывает текст из `humanizeError`) |
| info | `--copper` иконка | автоскрытие 4s |

Карточка `--ink-2`, border `--line`, прямые углы, текст `--paper`, мета `--mute`. Стек ≤3 видимых, остальные в очереди. `role="status"` (success/info) / `role="alert"` (error), `aria-live`. Пауза автоскрытия на hover. Свайп/× для закрытия. Без bounce — fade + translate-y 12px, 200ms.

```
┌────────────────────────────────────┐
│ ✓  Нарезка запущена                 │  success (gold ✓)
│    Готовые клипы появятся в Проектах │
└────────────────────────────────────┘
┌────────────────────────────────────┐
│▌✗  Не удалось сохранить профиль      │  error (--danger левый бордер)
│    Проверьте имя профиля   Подробнее │
└────────────────────────────────────┘
```

### 4.4 `humanizeError()` (закрывает FA3-02)

Сейчас сырые тех-ошибки утекают юзеру (`Ошибка 500: {"detail"...}`, `JSON.stringify` в 6+ местах). Чистая функция `humanizeError(error: unknown): { title: string; detail: string; hint?: string }`.

Логика:
- `ApiError` со `status` → маппинг на человеческий текст:
  - `400/422` → «Что-то с данными формы» + поле, если есть в `detail`.
  - `401/403` → «Нет доступа» + «проверьте ключ в Настройках».
  - `404` → «Не нашли — возможно, удалили или ссылка устарела».
  - `409` → «Конфликт: запись уже изменилась, обновите страницу».
  - `429` → «Слишком часто — подождите немного».
  - `5xx` → «Сбой на стороне приложения. Данные целы, попробуйте ещё раз.»
  - сеть/`Failed to fetch` → «Нет связи с сервером. Проверьте, запущен ли бэкенд.»
- Сырой `detail`/стектрейс → только в `hint`, и только в dev. В prod — никогда в UI.
- Используется тостами (§4.3), error boundary (§4.1) и инлайн-ошибками форм. Заменяет все `JSON.stringify`-места.

### 4.5 Деструктив-подтверждения (закрывает FA3-03 / 13× `window.confirm`)

`<ConfirmDialog>` примитив + `useConfirm()` промис-хук (замена нативного `window.confirm`, который ломает стиль и блокирует поток).

Модалка по брендбуку: оверлей `--ink/70 backdrop-blur`, карточка `--ink-2`, border `--line` (для деструктива — верхний бордер `--danger` 2px, точечно). Прямые углы. Заголовок display-serif, тело `--mute`. Кнопки: вторичная «Отмена» (`--mute-2`, ghost) + основная деструктивная (`--danger` текст/бордер, hover → заливка `--danger`).

```
┌──────────────────────────────────────┐
│▔▔▔▔▔▔▔▔▔▔▔▔▔▔▔▔▔▔▔ (--danger 2px) ▔│
│  Удалить проект «Интервью_03»?         │  display-serif
│                                       │
```

## РЕЗЮМЕ (для оркестратора)

**Структура навигации:** рейл сводится к 4 рабочим зонам (Студия / Проекты / Планировщик / Настройки) из единого словаря `routes.ts`; настройки перестают дублироваться в рейле — их единственный источник `SettingsSubNav` внутри `SettingsLayout`, теперь со всеми 8 разделами (вернули недостижимые `brand`/`maintenance`). Шапка по брендбуку: лого display-serif золотом + mono-крошки + sticky; рабочий drawer-адаптив сохранён 1:1.

**Переключатель режимов:** сегмент-контрол в шапке справа, глобальный, `aria-pressed`; состояние в `UiModeContext` (guided default, persist localStorage, no-flash чтение). Меняет степень раскрытия сложности (мастер vs форма-всё-сразу, инлайн-хинты vs hover-«?», человеческие vs тех-имена моделей, скрытые vs развёрнутые расширенные опции) — не функции и не роуты. Вводит недостающие `<Tooltip>`/`UiModeContext` (FA3-01).

**Системные паттерны (новый слой `components/ui/`):** (1) двухуровневый Error Boundary — root вокруг Outlet + route-level для lazy-чанков, экран «Клинок затупился» в самурайском стиле; (2) пустые состояния с CTA и оценкой времени; (3) тост-система success/error/info с очередью и `aria-live`; (4) `humanizeError()` — человеческие сообщения вместо сырых 500/`JSON.stringify`; (5) `ConfirmDialog`/`useConfirm` вместо 13× `window.confirm`; (6) скелетоны/лоадеры с одно-направленным shimmer и уважением `prefers-reduced-motion`. Всё на токенах брендбука (Kuro/Sumi/Hai база, Kinzoku gold-акцент, Chi `--danger` точечно), прямые углы, без запрещённых анимаций.
