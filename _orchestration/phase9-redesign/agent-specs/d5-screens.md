# D5 — Core Application Screens Redesign Spec

> Роль: Senior Product Designer (Core Application Screens). Душа: рилсы — герой, консистентность экранов, мобайл-адаптив.
> Язык: самурайский брендбук (10-website-style + 02-color-palette + 06-photo-style). Тёмная тема всегда, прямые углы, Sumi-карточки, золото-акцент, Noto Serif JP заголовки.

## 0. Design Language Foundation (общее для всех экранов)

### Токены (свести U-01/VD-06 в один словарь)

| Токен | HEX | Роль |
|---|---|---|
| `--kuro` | `#0A0A0A` | Фон страницы, первый слой |
| `--sumi` | `#1A1A1A` | Карточки, подложки, второй слой |
| `--hai` | `#2A2A2A` | Бордеры, разделители, тонкие линии |
| `--kinzoku` | `#C9A84C` | Главный акцент: CTA, активные состояния, заголовки-акценты |
| `--do` | `#B87333` | Вторичный: иконки, обводки, mono-теги |
| `--kogane` | `#E8C547` | Блик: hover-искра, 1-2 точки на экран |
| `--shiroi` | `#F0E6D2` | Основной текст |
| `--kasumi` | `#8A8278` | Мета: подписи, даты, плейсхолдеры, неактив |
| `--chi` | `#8B2500` | Деструктив (delete/nuke), точечно |

Пропорция на экране: тёмная база 65-75% / текст 15-20% / акценты 10-15% / chi 1-3%.

### Типографика

| Применение | Шрифт | Размер (desktop / mobile) |
|---|---|---|
| H1 экрана / hero | Noto Serif JP Bold | 36px / 24px |
| H2 секции | Noto Serif JP Bold | 22px / 18px |
| Body | Manrope | 17px / 15px (НЕ 11px — закрываем VD-05) |
| Мета / даты / статусы | JetBrains Mono | 12-13px |
| Микро-теги (uppercase) | Press Start 2P | 9-10px |

### Сетка и отступы

- max-width контента **1400px** (не 1200 — нужен под широкую галерею рилсов, закрываем VD-02).
- 12 колонок, gap 24px desktop; 4 колонки mobile.
- Отступы секций: 96px desktop / 48px mobile (плавная прогрессия, без скачка 120/60).
- Контейнер: `px-4 sm:px-6 lg:px-8`.

### Базовая карточка (Sumi-card) — единый примитив `<Card>`

```
border-radius: 0           прямые углы — всегда
background: --sumi (#1A1A1A)
border: 1px solid --hai
hover (pointer): border-color → --kinzoku, 0.3s
focus-visible: outline 2px --kinzoku, offset 2px
```

### Решения из аудита — общие принципы (применяются на КАЖДОМ экране)

| Проблема аудита | Решение в этом языке |
|---|---|
| **VD-03** действия скрыты `opacity-0 group-hover` → мертвы на тач | Действия карточек **всегда видимы** на coarse-pointer. Паттерн: на десктопе (`hover: hover`) вторичные действия приглушены до `--kasumi` и проявляются по hover; на тач (`hover: none`) — постоянная видимая mono-иконочная панель снизу карточки, таргет 44px. Primary-действие (открыть/like) видимо всегда везде. |
| **VD-02** узкая галерея | Reel-грид расширяется до `lg:grid-cols-4 xl:grid-cols-5 2xl:grid-cols-6` на 1400px+. Вертикаль 9:16 — карточки узкие, влезает много. |
| **44px touch-таргеты** | Все интерактивные элементы min `h-11 min-w-11` (44px) на тач. Иконки-кнопки получают невидимый hit-area паддинг. |
| **VD-04** модалки | Все модалки: `max-h-[85vh] overflow-y-auto`, sticky header с заголовком + close (44px), sticky footer с действиями. Sumi-фон, прямые углы, бэкдроп `--kuro/70`. |
| **FA3-02** сырые ошибки | Везде `humanizeError()` → человеческий текст + следующий шаг. Никаких `JSON.stringify` / «Ошибка 500». |
| **FA3-03** тосты вместо confirm | Деструктив → стилизованная `<ConfirmDialog>` (Sumi-модалка, chi-кнопка), результат → тост. |
| **2 режима** | `UiModeContext`: Пошаговый (новичок) / Эксперт. Переключатель в shell. Эксперт-only поля скрыты в Пошаговом за дисклоужерами. |

**Таймлайн:** `<PipelineTimeline>` (сохранён из аудита). Состояния узла: готово (`--kinzoku` ●), текущий (`--kogane` ◐ + пульс ≤1с, prefers-reduced-motion off), ждёт (`--kasumi` ○), ошибка (`--chi`). Текущая стадия — человеческим языком + счётчик + ETA.
**Cancel:** видимая кнопка «Отменить обработку» (chi-border, hover→chi-fill), только при `status=processing`. ConfirmDialog: «Остановить? Уже готовые рилсы сохранятся.» (честно — частичный результат остаётся).

**Hero-действия:** переименовать (инлайн), сменить vision-профиль (dropdown), удалить (soft/hard/nuke в подменю с ConfirmDialog, chi). Все тап-доступны.

**Статусы (mono + цвет):** ● Опубликовано (kinzoku) · ◐ В очереди (kogane) · ○ Ждёт (kasumi) · ⚠ Не отправилось (chi).
**Cancel — честность (закрываем fiction #4):** «Снять» с запланированного назначения. Если пост ещё в очереди Publer — снимаем реально. Если уже опубликован — кнопка не «Снять», а «Открыть пост» (снять опубликованное нельзя; не обещаем ретракт). ConfirmDialog поясняет.
**Connections — честность (закрываем fiction #5):** НЕТ страницы `/settings/connections` с мёртвым YouTube-OAuth. Подключение аккаунтов идёт через Publer; в мастере неподключённая площадка показывает «Добавить в Publer ↗» (внешняя ссылка), не фейковую кнопку Connect.
**Компоненты:** `<CampaignCard>`, `<CampaignWizard>` (4 шага), `<AssignmentRow>`/`<AssignmentCard>` (адаптив таблица↔карточки), `<StatusBadge>`, `<AccountPicker>`, `<CaptionPresetSelect>`.

**Режимы (закрываем U-03/FA3-01):** Пошаговый прячет 90% тумблеров, показывает осмысленные пресеты («Динамичные короткие» / «Спокойные длинные» / «Авто»). Эксперт раскрывает все ~25 групп через дисклоужеры.
**Tooltip везде:** каждый control имеет `[?]` с человеческим пояснением (settings-примитивы с обязательным `hint` сохранены из аудита).
**Честность настроек (closes fictions #2, #3):**
- «Режим нейросети» (LLM tier): явная подпись, что все варианты = одна быстрая модель; это про скорость, не про качество. Не обещаем «pro inference».
- `chaptered` режим: помечен ⚠ «ненадёжно на монологах», но оставлен.
- mouth_sound_removal / screencast-zoom / deictic-zoom: остаются скрытыми (dormant) — не показываем нерабочее.
**Машинные имена:** `gemini/zhipu` → человеческие лейблы в селектах (закрываем FA3-04).
**Компоненты:** `<SettingsLayout>` (единая sub-nav), `<SettingGroup>` (disclosure), `<Toggle>`, `<RadioGroup>`, `<Slider>`, `<Tooltip>`, `<Select>` (человеческие лейблы). Все ≥44px на тач.

## Сводка: как закрыты проблемы аудита

| Проблема | Где и как решена |
|---|---|
| **VD-02** узкая галерея | Экран 2: reel-грид `lg:4 / xl:5 / 2xl:6`, max-width 1400px. |
| **VD-03** hover-only действия | Все экраны: primary-действия видимы всегда; на тач (`hover:none`) — постоянная иконочная панель 44px; hover-проявление только как десктоп-украшение поверх уже доступного. Tinder — 3 явные кнопки дублируют свайп. |
| **VD-04** модалки | ExportDialog, ProjectFormModal, ConfirmDialog: `max-h-85vh`, overflow-y-auto, sticky header/footer, close 44px. |
| **VD-05** мелкий текст | Body 15-17px везде (не 11px). Mono 12-13px только для меты. |
| **44px touch** | Все интерактивы `min-h-11 min-w-11` на тач; иконки-кнопки — невидимый hit-area. |
| **U-01/VD-06** двойные токены | Единый Sumi-словарь (раздел 0). |
| **U-02** дубль навигации | Экран 7: одна «Настройки» sub-nav, без дублей в NavRail. |
| **U-03** форма-монстр | Экраны 1 и 7: Пошаговый/Эксперт; визард = шаги; настройки за дисклоужерами. |
| **FA3-01** нет режимов/тултипов | UiModeToggle в shell; `[?]` Tooltip на каждом control настроек. |
| **FA3-02** сырые ошибки | `humanizeError()` везде. |
| **FA3-03** confirm/тосты | `<ConfirmDialog>` (Sumi/chi) + тосты на результат. |
| **FA4-P0** error boundaries | Экран 6 и shell: Error Boundary + Suspense-fallback с retry. |
| **Честность (exposure fictions)** | Скор = открытая эвристика с тултипом (не фейк-AI-вес); экспорт = «отдаём MP4 как есть» без косметич. битрейта; cancel-джоб реален с сохранением частичного результата; cancel-назначения снимает только из очереди, опубликованное не «ретрактит»; нет dead YouTube-OAuth страницы (подключение через Publer); LLM-tier честно описан как про скорость; chaptered помечен ⚠. |

**Новые/единые UI-примитивы под весь редизайн:** `<Card>`, `<Button>` (primary kinzoku-fill / secondary kasumi-border / destructive chi-border), `<Input>/<Textarea>`, `<Modal>`+`<ConfirmDialog>`, `<Tooltip>`, `<StatusBadge>`, `<ProgressBar>`, `<FilterTabs>`, `<ModeToggle>`, `<Toast>`. Все — прямые углы, Sumi-фон, kinzoku-фокус-ринг, prefers-reduced-motion-aware.
