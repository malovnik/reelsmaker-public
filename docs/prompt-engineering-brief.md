# Prompt Engineering Brief — Deep Chain-of-Thought Role Prompts

Этот документ — общий контекст для subagent'ов, пишущих deep-role промпты для 13
стадий пайплайна videomaker. Каждый subagent получает стадийно-специфичный брифинг
+ указание прочитать этот файл для общего контекста.

---

## Зачем

Текущие промпты (300-2000 chars) слишком поверхностны. Gemini 2.5 работает сильно
лучше на deep Chain-of-Thought role prompts (10-15k chars), задающих identity,
ментальную модель, пошаговую процедуру, критерии качества, failure modes и жёсткие
констрейнты. Few-shot примеры **запрещены** пользователем — они размывают
роль и провоцируют слепое копирование поверхностных паттернов.

---

## Выходной формат (жёсткий)

Каждый deep-role промпт — **ровно 10000-15000 символов** UTF-8 plain text на русском
языке (technical terms в англ OK: JSON, timestamp, embedding, diarization, chunk,
speaker, hook, payoff). Структура из 7 секций в фиксированном порядке:

```
=== IDENTITY ===
<500-1000 символов>

=== MENTAL MODEL ===
<2500-3500 символов>

=== DECISION PROCEDURE ===
<2500-3500 символов>

=== QUALITY CRITERIA ===
<1500-2500 символов>

=== FAILURE MODES ===
<800-1200 символов>

=== CONSTRAINTS ===
<400-600 символов>

=== OUTPUT SCHEMA ===
<JSON schema verbatim из брифа стадии + директива "JSON-only output, no markdown">
```

**Первая строка** промпта — `=== IDENTITY ===`. Без преамбулы, без markdown-обёртки,
без извинений. **Последняя строка** — финал секции OUTPUT SCHEMA.

## Запреты (hard rules)

- **НЕТ few-shot примеров.** Ни "strong example", ни "weak example", ни иллюстративных
  цитат, ни sample inputs/outputs. Оперируй признаками, диапазонами, механизмами.
- **НЕТ эмодзи.**
- **НЕТ клише:** "помни", "никогда не забывай", "крайне важно", "по-настоящему
  важно", "давайте рассмотрим", "на самом деле".
- **НЕТ smalltalk'а:** "пожалуйста", "если хочешь", "не стесняйся".
- **НЕТ markdown code fences** вокруг финального текста.
- **НЕТ текста вне 7 секций** (ни вступления, ни заключения, ни "Вот промпт:").
- **НЕТ второго лица с softener'ами:** только прямой императив.

---

## Стиль

- Крепкий технический русский на уровне senior-engineer'а.
- Императив без размягчений.
- Буллеты только когда контент действительно перечислим.
- Короткие абзацы (3-6 строк).
- Числовые границы — цифрами ("0.65", "3-15 секунд", "31-89s").

## Картозия — фреймворк драматургии (применяй где релевантно)

- **Сверхидея** — философская ось проекта в одном предложении. Не слоган, не тезис,
  а онтологическая рамка. Картозия в интервью говорит: "Орёл и решка" держался
  на оппозиции "один в пути с золотой картой vs один на 100 долларов"; сюжеты
  Навального о коррупции — на "фобия потери как двигатель успеха".

- **Одна арка = одна мысль.** Если начали про пробки — заканчиваем про пробки.
  Две темы в одном рилсе — сигнал "рассыпанное удержание". Это железный закон
  Картозии.

- **Book-end symmetry.** Финал эхом перекликается с хуком через конкретный мотив —
  формулировка, образ или callback. Без симметрии рилс ощущается оборванным:
  "смотрел-смотрел — а зачем?".

- **Эмоциональные качели.** Контраст обязателен: препятствие → сопереживание →
  катарсис. Цепочка "смех → стыд → восхищение" — тот же принцип. Монотонная
  эмоция убивает retention.

- **Dramatic irony.** Момент, когда зритель знает больше участников, даёт ему
  ощущение превосходства — мощный удерживающий крючок. Картозия: "Приоткройте
  зрителю тайны — привяжете к контенту".

- **Парадокс как генератор хука.** Абсурдное сравнение, конфликтный факт,
  внутренне противоречивое утверждение — это диссонанс, на который мозг не
  может не среагировать.

- **Semantic closure.** Payoff — не произвольный финал, а finishing move: она
  закрывает open-loop-вопрос хука, содержит грамматически завершённое
  предложение + emotional_beat уровня triumph/relief/reveal. Payoff на role=
  development — это обрыв мысли, не замыкание.

- **Цитируемость.** Фраза, которую хочется сохранить и переслать, — сильный
  маркер. Отсутствие цитируемых единиц в кандидате — сигнал слабости.

## OpusClip research (AAAI-2025, arXiv:2412.08879, Repurpose-10K)

- **Virality Score = 4 измерения.** Hook (первые 3-5 секунд), Flow (связность
  + satisfying conclusion), Value (плотность + резонанс), Trend (попадание в
  актуальные темы аудитории). Каждое измерение — отдельный сигнал, не сводятся
  друг к другу.

- **Joint classification + regression.** OpusClip-модель одновременно
  классифицирует каждый segment (брать/не брать) и регрессирует точные
  start/end timestamps — обучена на 120 тысячах аннотаций с критерием
  "satisfying conclusion".

- **Boundary regression signals.** Satisfying conclusion распознаётся по
  совокупности: завершённое предложение ASR + падение audio energy + пауза
  >0.8s после тезиса + маркёры "...и поэтому...", "...так и вышло...", смех.

- **Narrative completeness > duration.** Длина клипа — производная от того,
  сколько нужно для замыкания мысли. 22 секунды если достаточно, 87 если нет.
  Фиксированный target по длительности — антипаттерн.

- **Non-contiguous assembly.** Premise и payoff могут браться из разных
  временных точек видео. Связка — через стабильный motif anchor (см.
  motif_tracker).

- **Genre weighting.** Q&A / vlog / webinar / educational / listicle /
  interview — разные комбинации весов Hook/Flow/Value/Trend. Для Q&A выше
  Hook+Flow; для educational — Value; для vlog — Hook+Trend. Не ветви
  процедуру, но учитывай в критериях.

- **Open Loop на Hook.** Hook без незакрытого контракта со зрителем (вопрос,
  на который ответ будет позже) не проходит фильтр Virality Score > 60.

## Как использовать этот файл

Subagent перед написанием промпта:
1. Читает этот файл (общий контекст).
2. Читает стадийно-специфичный брифинг из текста задачи.
3. Пишет deep-role промпт по 7-секционной структуре.
4. Возвращает ТОЛЬКО текст промпта — без комментариев, без markdown.

Конечный текст попадёт в `apps/backend/src/videomaker/services/prompts.py`
как строка-константа и будет передаваться Gemini 2.5 как system_instruction.
