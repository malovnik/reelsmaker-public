# Reelibra — запуск на macOS

Требуется **Mac с Apple Silicon (M1 и новее)** и **macOS 13+** (рекомендуется 14/15).
На Intel-Mac локальная транскрибация (MLX) не работает — нужен облачный ключ Deepgram,
лаунчер честно об этом предупредит.

## Как запустить

1. Скачайте и распакуйте релиз.
2. **Двойной клик по `reelibraMAC.command`** в корне папки.

При первом запуске в окне Terminal пойдёт нумерованный прогресс:

```
[1/5] Проверка платформы
[2/5] Проверка и установка инструментов   ← uv, Python 3.12, Node 20, ffmpeg
[3/5] Зависимости проекта                 ← uv sync + pnpm install
[4/5] Проверка и чистка прошлого запуска
[5/5] Запуск                              ← открывает http://localhost:3000
```

Первый запуск занимает несколько минут (скачиваются интерпретатор, Node, ffmpeg,
зависимости и ML-модели). Последующие — секунды. Всё кладётся локально в
`.reelibra-runtime/` внутри папки проекта — система не трогается, Homebrew не нужен.

## Gatekeeper («неизвестный разработчик»)

Лаунчер не подписан Apple Developer ID, поэтому при первом открытии скачанного
файла macOS покажет предупреждение. Это нормально для open-source без платного
аккаунта. Откройте **один раз** так:

- **right-click (Ctrl+клик) по `reelibraMAC.command` → Open → Open.**

После первого подтверждения macOS запомнит разрешение, дальше — обычный двойной клик.

Лаунчер сам снимает quarantine со своих скриптов и скачанных бинарей
(`xattr -dr com.apple.quarantine`) и ad-hoc подписывает нативные бинарники
(Node, ffmpeg) — на Apple Silicon без этого они падают с «killed: 9».

## Опционально: .app с иконкой

Если хотите нативную иконку и вид приложения вместо `.command`:

```bash
bash launchers/macos/make-app.sh
```

Соберётся `reelibraMAC.app` с иконкой `assets/reelibra.icns` и ad-hoc подписью.
Двойной клик по нему открывает Terminal с тем же прогрессом. Первый запуск —
так же через right-click → Open.

## Ключи

После старта впишите `GEMINI_API_KEY` в Настройках (или в `.env`) — без него
нарезка не работает. Файл `.env` создаётся автоматически из `.env.example`.

## Что делает лаунчер при каждом запуске

- Проверяет/доустанавливает uv, Python 3.12, Node 20, ffmpeg (идемпотентно).
- `uv sync` + `pnpm install` (быстро, если ничего не менялось).
- Гасит зависшие процессы прошлого запуска (uvicorn/vite/esbuild/ffmpeg),
  graceful `SIGTERM` → 5с → `SIGKILL`, освобождает порты 8000 и 3000.
- Удаляет мусор `.partial`/`.tmp`/orphan-`.lock`. **Базу и видео не трогает** —
  prerванные задачи backend сам помечает как ошибку на старте.
- Поднимает backend :8000 + frontend :3000, ждёт готовности, открывает браузер.

Логи: `data/logs/bootstrap.log`, `data/logs/backend.log`, `data/logs/frontend.log`.
